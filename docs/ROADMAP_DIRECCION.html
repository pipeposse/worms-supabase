Proyecto WORMS · Plan de implementación

De Excels desordenados a una plataforma de producción confiable

Reemplazo del sistema actual de planillas por una base de datos centralizada, una app web para carga validada y trazabilidad completa de quién carga qué. Cero servidor propio, cero costos recurrentes.

1. ¿Por qué ahora?

⚠️ Hoy

  • Cada sector tiene su Excel propio, con formato distinto
  • No hay forma de saber quién cargó cada dato
  • Errores tipográficos y unidades inconsistentes (kg vs tn vs litros)
  • Si una planilla se corrompe o se pisa, no hay recuperación
  • Conciliar al cierre de mes lleva horas y deja desfasajes
  • Dashboards desactualizados y manuales

✅ Después

  • Una sola base de datos en la nube (Supabase, gratis)
  • Login obligatorio: cada carga firmada con el id del operador
  • Formularios con validación: sin texto libre donde no corresponde
  • Backup automático diario
  • Conversión automática de unidades a TN
  • Alertas de parámetros fuera de rango en tiempo real

2. ¿Cómo funciona, en 30 segundos?

El operador entra a una página web local (o con URL pública si decidimos deployar a la nube). Se loguea con su usuario y un PIN de 4 dígitos. Carga su análisis o batch en un formulario que no le permite ingresar valores fuera de rango ni unidades inconsistentes. Cada carga queda firmada con su id. El supervisor ve todo en tiempo real.

7. Trazabilidad y compliance

Cada acción que hace cualquier usuario en la plataforma queda registrada de forma inmutable, con su id, fecha, hora y el detalle del cambio.

  • Quién: id de usuario logueado, no texto libre — imposible falsificar
  • Cuándo: timestamp con segundos
  • Qué: tabla y registro afectado
  • Cambio: diff entre el valor anterior y el nuevo, en JSON
  • Vista de auditoría: el supervisor ve un panel con los últimos 100 eventos en tiempo real

Si una autoridad regulatoria pide trazabilidad de un lote, podemos contestar en 5 minutos.

8. Decisiones que necesitamos confirmadas

8.1 ¿Aprobamos el uso de Supabase (datos en cloud externo)?
Pro: gratis, backups, accesible desde cualquier lado · Con: datos viven fuera del servidor propio. Alternativa: instalar PostgreSQL en servidor local (requiere tiempo de IT y máquina prendida 24/7).
8.2 ¿La app vive en la red interna o en URL pública?
Interna: solo accesible desde la fábrica · Pública (Streamlit Cloud): accesible desde casa, gratis, protegida con login obligatorio. Recomendamos la pública.
8.3 ¿Migramos el histórico o empezamos desde cero?
Migrar el último trimestre tarda ~3 horas y deja la app con datos comparables. Empezar limpio es más rápido pero los reportes de los próximos meses no tienen referencia previa.
8.4 ¿Desactivamos el Excel viejo después del launch?
Recomendado: dejarlo en modo lectura por 30 días. Después se archiva. Si se mantiene editable, los operadores pueden volver al Excel y romper la consistencia.
8.5 ¿Quién es el dueño funcional del catálogo de productos / parámetros?
Propuesta: Eugenia es la que aprueba altas y cambios. Felipe los ejecuta en la base.
